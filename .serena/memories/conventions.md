# Conventions

- Antes de alterar arquivos, informar quais arquivos serao criados/editados.
- Nao implementar codigo funcional completo sem pedido explicito do usuario.
- Manter solucoes simples o suficiente para apresentacao academica.
- Explicar cada arquivo e partes importantes do codigo em linguagem de iniciante.
- Em caso de duvida relevante, perguntar em portugues com 2 a 4 opcoes e aguardar autorizacao antes de seguir.
- Frontend inicial sem React; backend inicial sem banco profissional.
- Sistema web/backend deve permanecer com 5 vagas; nao adaptar para menos vagas por causa do estado temporario do Wokwi.
- Frontend deve renderizar vagas dinamicamente a partir da API, evitando cards fixos no HTML.
- Todo frontend criado deve ser responsivo para celular e computador.
- Catraca deve permanecer simples: botao no frontend, comando no FastAPI, estado salvo no JSON e fechamento automatico apos 5 segundos.
- Nao implementar atualizacao em lote sem autorizacao explicita.
